# Vi-MTPro (Cascade + WARP)

Быстрая установка MTProxy с поддержкой каскада через WARP.

Два режима работы:
- **Одиночный** — один VPS. По желанию трафик в Telegram идёт через локальный Cloudflare WARP (mtg → `socks5://127.0.0.1:40000` → WARP)
- **Каскад** — Первый-сервер → Второй-сервер → WARP → Telegram (обход блокировок)

В одиночном режиме при установке скрипт может настроить `Managed fronting domain`: скрипт проверит, что домен уже резолвится в IP сервера, затем автоматически поднимет `nginx` + Let's Encrypt на `443`. Включить/выключить локальный WARP можно позже: `vi-mtpro` → Управление → «Локальный WARP».

## Обфускация

Трафик маскируется под обычный HTTPS и устойчив к DPI и active-probing:

- **Managed fronting domain** — отдельный режим. Ваш домен привязывается к серверу, под него поднимается HTTPS на `443`, и managed-клиенты используют этот домен как SNI.
- **External FakeTLS (секрет `ee`)** — отдельный режим. Клиент использует внешний популярный домен как SNI/FakeTLS-маску, без требования поднимать этот домен локально на вашем сервере.
- **Случайный порт** — по умолчанию выбирается случайный порт из типичного для HTTPS диапазона (`443`, `8443`, `2053`, `2083`, `2087`, `2096`), но для `Managed fronting` клиент не использует `443` как bind-порт, потому что `443` зарезервирован под локальный HTTPS/`nginx`.
- **Анти-DPI защиты mtg** — `anti-replay` (защита от сканеров-пробников), `doppelganger` с динамическим размером TLS-записей (рандомизация паттернов), сглаживание временного фингерпринта.

## Быстрый старт

### Первый-сервер (основной)

```bash
bash <(curl -sSL https://raw.githubusercontent.com/ViPunch/Vi-MTProxy/master/setup.sh)
```

При первом запуске выберите режим — одиночный или каскад. Затем можно:

- настроить `Managed fronting domain`, который уже резолвится в IP сервера;
- или использовать `External FakeTLS domain` из списка популярных доменов или ввести его вручную.

Скрипт установит mtg, при необходимости поднимет `nginx` + Let's Encrypt, создаст первого клиента и выдаст ссылку для Telegram.

После установки управляйте прокси одной командой:

```bash
vi-mtpro
```

---

### Второй-сервер (только для режима каскада)

```bash
bash <(curl -sSL https://raw.githubusercontent.com/ViPunch/Vi-MTProxy/master/tunnel.sh)
```

Скрипт установит WARP и gost, поднимет SOCKS5-туннель на порту 1080. После этого вернитесь на Первый-сервер — каскад заработает.

## Что умеет setup.sh

- Настраивать `Managed fronting domain` с автоматическим HTTPS на `443`
- Добавлять клиентов двух явных типов: `Managed fronting` и `External FakeTLS`
- Показывать готовые `tg://` ссылки для всех клиентов
- Автоматически прогонять `mtg doctor` после создания клиента
- Привязывать и отвязывать EU-сервер без пересоздания клиентов
- Обновлять mtg до последней версии
- Удалять отдельных клиентов или всё целиком

## Режимы клиента

- **Managed fronting**
  - использует ваш собственный домен, уже указывающий на сервер;
  - требует локальный `nginx` и доступный `443/tcp`;
  - не использует `443` как клиентский bind-порт.

- **External FakeTLS**
  - использует внешний домен как TLS-маску;
  - не требует, чтобы этот домен указывал на ваш сервер;
  - не требует локальный `nginx` именно для этого клиента.

## Что умеет tunnel.sh

- Устанавливать WARP и gost одной командой
- Показывать статус туннеля
- Полностью удалять туннель
